Radio-first message transport

Short encrypted messages over analog audio channels

FSK Comm is a Java 17 stack that turns logical messages into BFSK audio bursts and recovers them back into validated payloads. It is built for compact field messaging, reproducible desktop diagnostics, and Android-style record-then-decode flows.

Overview

The project focuses on a narrow problem: reliable transport of short messages through analog audio, with encryption, fragmentation, frame protection, structured diagnostics, and a clean boundary between protocol logic and host I/O.

Secure by design

Authenticated encryption

ChaCha20-Poly1305 protects message payloads, while replay protection guards against duplicate accepted messages.

Channel resilience

Protected PHY frames

CRC, whitening, FEC, interleaving, burst search, and frame recovery are part of the active implementation.

Host ready

Desktop and Android paths

The desktop CLI covers debug workflows, while the runtime module is shaped for Android integration.

Architecture

core

Logical messages, encryption, replay protection, fragmentation, reassembly, and diagnostics contracts.

phy

Transmission profiles, frame codec, BFSK modulation, demodulation, burst search, and PCM helpers.

runtime

Encode and decode orchestration for record-then-decode sessions and mobile-friendly integration.

app

CLI tooling for WAV inspection, metadata dumps, profile comparison, and verbose diagnostics.

Recommended usage today

  • Use the robust profile as the default choice for real radio conditions.
  • Prefer record-then-decode as the main mobile workflow.
  • Keep audio I/O in the host application and modem/message logic inside runtime.
  • Use structured diagnostics instead of treating CLI text as the source of truth.
